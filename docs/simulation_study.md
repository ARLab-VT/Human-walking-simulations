You are working as a research software engineer on a neuromusculoskeletal disturbance-recovery project based on the official `amathislab/musclemimic` repository.

## Research objective

Build an extensible MuscleMimic study that starts from a released pretrained MyoFullBody walking policy and augments it with:

1. A low-dimensional, spinal-feedback-inspired reflex residual.
2. A small learned recovery policy.
3. A continuous recovery-priority objective that relaxes motion imitation and speed tracking during instability.
4. Configurable erroneous exoskeleton torque impulses at the bilateral hip, knee, and ankle.
5. Scientific evaluation tools for measuring phase-specific and joint-specific disturbance recovery.

The intended controller is:

[
u_{\mathrm{cmd}}
================

\operatorname{clip}
\left(
u_{\mathrm{base}}
+
\alpha,u_{\mathrm{reflex}}
+
\beta,u_{\mathrm{recovery}}
\right)
]

where:

* (u_{\mathrm{base}}) is generated by the frozen pretrained MuscleMimic policy.
* (u_{\mathrm{reflex}}) is an explicit grouped reflex correction.
* (u_{\mathrm{recovery}}) is generated by a small trainable residual policy.
* (\alpha) is a configurable reflex scale.
* (\beta\in[0,1]) is a continuous instability/recovery activation.
* All actions must obey the true actuator control ranges and rate limits.

Do not assume that the MuscleMimic action represents raw muscle excitation. Inspect the repository and loaded MuJoCo model to determine exactly what the policy action means, where it is transformed, and where the safest residual-injection point is.

## Core engineering principles

1. Preserve the upstream MuscleMimic implementation wherever possible.
2. Put reusable code in Python modules, not only in notebook cells.
3. Use the notebook as an executable research narrative that imports those modules.
4. Add new configurations rather than changing upstream default configurations.
5. Freeze the pretrained base policy during initial recovery-policy training.
6. Use `jax.lax.stop_gradient` or the repository’s equivalent to guarantee that base-policy parameters do not update.
7. Keep all new logic JAX-compatible and vectorized.
8. Avoid Python loops inside JIT-compiled environment steps.
9. Keep perturbations, reflex gains, reward weights, delays, and thresholds configurable.
10. Add unit tests and smoke tests at every stage.
11. Do not silently change the physics timestep, policy frequency, actuator model, contact parameters, or reference-motion processing.
12. Record all assumptions in Markdown and configuration comments.
13. Do not claim physiological validity merely because muscles are present. Label the controller “spinal-feedback-inspired” until validated against human data.
14. Do not embed Hugging Face or W&B tokens in source files or notebooks.
15. Do not start a long training job until the baseline, reflex, perturbation, and short residual-training smoke tests pass.
